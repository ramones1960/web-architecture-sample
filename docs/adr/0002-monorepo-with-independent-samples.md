# ADR-0002: モノレポ + サンプル自己完結方針

- **Status**: Accepted
- **Date**: 2026-06-15
- **Deciders**: リポジトリメンテナ

## Context（背景）

複数言語（TypeScript / Go / Python / Java など）にまたがる多様な
Web アーキテクチャサンプルを提供したい。各サンプルは比較・学習・コピー利用
されることを想定しており、相互に影響し合わずに独立して進化できる必要がある。
一方で、カタログ・運用ルール・テンプレートは横断的に共有したい。

## Decision（決定）

1. **モノレポ** を採用し、全サンプルを 1 リポジトリの `samples/` 配下に集約する。
2. 各サンプルは **自己完結（self-contained）** とする。
   依存・ビルド・テスト・実行手順をサンプルディレクトリ内に閉じ、
   サンプル間でコードを直接共有しない。
3. ルートには **特定言語のツールチェーンを置かない**（package.json 等で汚さない）。
   ルートはカタログ・規約・ドキュメント・運用スクリプトのみを担う。
4. 分類は `samples/<category>/<architecture-name>/` の 2 階層とする。

## Consequences（結果・トレードオフ）

- ✅ ポリグロットに対応でき、サンプルごとに最適な技術を選べる。
- ✅ サンプル単位で独立して追加・更新・廃止できる（疎結合）。
- ✅ 1 リポジトリで横断的な比較・検索・索引が可能。
- ⚠️ コードの共有はできない（DRY ではない）。
  共有したい知見はコードではなくドキュメントで伝える方針とする。
- ⚠️ ルートに統一ビルドツールがないため、CI は「変更サンプル検出 + 各サンプルの
  自前タスク実行」という形を取る。

## Alternatives（検討した代替案）

- **言語ごとの個別リポジトリ（マルチレポ）**: 横断比較・索引がしにくく、
  運用ルールが分散するため不採用。
- **pnpm/Turborepo 等の単一ビルドツールでの統合**: TS/JS には有効だが
  ポリグロット要件に反するため、ルートでは採用しない
  （個別サンプル内で採用するのは自由）。
- **共有パッケージ層 (`packages/`) を設ける**: サンプル間結合を生み、
  独立進化を阻害するため現時点では設けない。必要になれば ADR で再検討する。
