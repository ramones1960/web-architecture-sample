# ADR-0001: ADR を導入して意思決定を記録する

- **Status**: Accepted
- **Date**: 2026-06-15
- **Deciders**: リポジトリメンテナ

## Context（背景）

本リポジトリは多様な Web アーキテクチャのサンプルを継続的に追加していく。
「なぜこの構造・技術・方針を選んだのか」が時間とともに失われると、
後からの参加者が判断の前提を理解できず、過去の議論を繰り返すことになる。

## Decision（決定）

アーキテクチャ・運用上の重要な決定を **ADR（Architecture Decision Record）** として
`docs/adr/` に記録する。形式は Michael Nygard 形式に準拠し、
連番ファイル名 `NNNN-<title>.md` で管理する。

## Consequences（結果・トレードオフ）

- ✅ 決定の背景と代替案が残り、判断を追跡できる。
- ✅ レビュー時に「なぜ」を議論する共通の場ができる。
- ⚠️ 記録のための軽い運用コストが発生する（重要な決定に限定して緩和する）。

## Alternatives（検討した代替案）

- README に散文で書く: 構造化されず、決定の履歴が追えないため不採用。
- Wiki / 外部ツール: コードと乖離しやすいため不採用。リポジトリ内に置く。
